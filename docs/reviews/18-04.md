## 2. Decisiones y trade-offs técnicos (para revisitar)

### 2.1 · AES_KEY vía env var (no KMS)
- **Estado actual:** la clave vive en `Settings.aes_key` cargada de `.env`.
- **Riesgo:** comprometer el host compromete todas las biomarker rows.
- **Upgrade path para producción:** AWS Secrets Manager / GCP Secret Manager con rotación. La API del helper (`encrypt_biomarker`/`decrypt_biomarker`) soporta swap sin tocar llamadores.
- **Prioridad:** alta antes de aceptar PII real.

### 2.2 · HITL threshold de 0.7 sin calibrar
- Umbrales en `entity_resolution.py`: REJECT=0.6, HITL=0.7. Son **heurísticas iniciales**.
- **Plan de calibración:** generar 200+ pares `(extracted_name, canonical_name)` ground-truth (mezcla de OFF + etiquetas reales de MX/LatAm). Métricas: precision/recall por bucket de confidence. Ajustar hasta precision@HITL ≥ 0.95.
- **Dataset sugerido:** exportar 300 `scan_history` rows una vez en producción; re-etiquetar manualmente.

### 2.3 · BGE-M3 fallback: solo stub
- `services/embeddings.py::_embed_local_bge` lanza `NotImplementedError`.
- **Razón:** `sentence-transformers` + torch añade ~500 MB a la imagen.
- **Cuándo activarlo:** cuando (a) Gemini tenga outage sostenido, o (b) queramos independencia de API. Requiere re-indexar Chroma (768→1024 dims).

### 2.4 · Ingesta real vs seed curado
- `scripts/seed_rag.py --live` existe pero **no está probado contra URLs reales**. La corrida por defecto usa `data/seed/additives.json` (20 aditivos curados, FDA+EFSA+Codex simulados).
- **Pendiente validar live:**
  - FDA EAFUS URL `https://www.cfsanappsexternal.fda.gov/scripts/fdcc/eafus.xlsx` (cambia con frecuencia).
  - EFSA Zenodo `record/8120114` — confirmar que el schema JSON mapea correctamente a `parse_records`.
  - Codex GSFA scraping — endpoint `fao.org/gsfaonline/additives/results.html` puede cambiar estructura.
- **Estrategia defensiva:** agregar `ingestion_log.status = 'PARTIAL'` si un batch falla parcialmente; idempotencia ya garantizada por upsert.

### 2.5 · Chunk size 512 tokens vs embedding template 256
- Resuelto en `docs/embedding-strategy.md`: chunk fuente (persistido en Chroma `document`) = 512 tokens; el **embedding template** (texto normalizado que se vectoriza) = ~256 tokens.
- Son artefactos distintos.

### 2.6 · BM25 con `BM25L` (no Okapi)
- Con corpus chicos (<100 docs), BM25Okapi devuelve 0 para todos los scores cuando IDF colapsa. Usamos `BM25L` que maneja mejor corpora pequeños.
- **Cuándo revisitar:** cuando el corpus pase de ~10k docs (ingesta real), reevaluar Okapi vs L vs Plus.

### 2.7 · Ruta `/scan/photo` sin rate limit de tamaño de imagen
- Endpoint valida 10 MB máx en `services/gemini._decode_image`. 413 al exceder.
- **Pendiente:** rate limit específico por bytes/min para prevenir abuso de Gemini API.

### 8.3 · Nuevos trade-offs descubiertos

#### 8.3.1 · slowapi 0.1.9 incompatible con `async def` + `from __future__ import annotations`

Síntoma: FastAPI devuelve `422 Unprocessable Entity` con `"loc":["query","body"]` en endpoints async decorados con `@limiter.limit(...)` cuando el módulo declara `from __future__ import annotations`.

Causa: el wrapper de slowapi sobre funciones async interactúa mal con la lazy evaluation de anotaciones — FastAPI no resuelve el forward-ref `"BarcodeRequest"` y trata al parámetro como query en vez de body.

**Fix aplicado:** quitar `from __future__ import annotations` en `routers/scan.py` (no afecta al resto de módulos). Tests verdes después.

**Deuda:** si alguien añade el import por costumbre, las pruebas fallan silenciosamente en producción porque el rate-limit bypass no salta. Mitigación futura: upgrade a slowapi >=0.1.10 (si sale) o middleware de limiter dedicado que no dependa del decorador por-endpoint.

#### 8.3.2 · Entity resolution: falsos positivos en labels en idiomas con poco corpus

Evidencia del smoke test real con Nutella (label en francés):
```
'Sucre' → canonical=Sucralose confidence=0.71
```
`Sucre` (azúcar en francés) machea por `token_sort_ratio` con `Sucralose` (edulcorante) con score 71/100, justo arriba del umbral HITL 0.7. Son moléculas distintas.

**Causa:** el seed `additives.json` tiene `Sucralose` pero no `Sucre` ni `Sucrose` como synonym; el fuzzy matcher encuentra la raíz común "sucr" y dispara.

**Mitigación corta:** añadir synonyms multi-idioma a los ingredientes más comunes (azúcar/sucre/sugar/sucrose → sacarosa E1200 o etiquetar como "Sucrose"). Tracked como ítem de calibración §2.2 de este doc.

**Mitigación larga:** ampliar el umbral REJECT a 0.75 para queries >4 chars cortas y/o filtrar candidatos por coincidencia de prefijo en el primer token.

### 9.3 · Trade-off nuevo: EFSA embedding en free tier de Gemini

**Síntoma:** con 4595 registros EFSA y free tier (100 req/min ≈ 15 RPM efectivos), el seed `--live` tarda 30-60 min o más, agotando la quota del proyecto antes de terminar.

**Causa raíz:** `index_record` llama `embed_text` una vez por registro de forma secuencial. 4595 registros × 4s delay = ~18 min mínimo; los retries por quota lo multiplican significativamente.

**Solución corto plazo:**
- Usar `--live` solo desde staging con API key de tier pagado, fuera de horario de desarrollo activo.
- En dev local: `python -m scripts.seed_rag` (curated, 31 registros, sin rate limit).

**Solución largo plazo (pendiente):**
1. Separar SQL upsert del embedding en `index_record`: persistir primero, embedar en batch asíncrono posterior.
2. Flag `--embed-limit N` en el seed para limitar registros embebidos por corrida.
3. Endpoint `/admin/embed-pending` que embeda registros SQL sin vector en Chroma (cron-safe).

### 9.4 · Nuevos trade-offs técnicos

#### 9.4.1 · EFSA Zenodo: schema era Excel, no JSON
El parser anterior buscaba `data`/`records` en el JSON de la API de Zenodo, pero la respuesta es metadata de registro con links a archivos `.xlsx`. Reescrito para descargar `SubstanceCharacterisation_KJ_2023.xlsx` directamente vía `files[].links.self`.

#### 9.4.2 · FDA EAFUS: URL muerta indefinida
FDA no publica changelog de endpoints. Se implementó `CANDIDATE_URLS` con validación por magic bytes (`PK` = zip/xlsx). Cuando FDA publique una nueva URL, agregarla en `fda_eafus.py:CANDIDATE_URLS`.

#### 9.4.3 · `ALLOWED_ORIGINS` requiere JSON array en pydantic-settings
Un campo `list[str]` en `.env` debe estar en formato JSON (`["a","b"]`), no CSV (`a,b`). Error silencioso: pydantic lanza `SettingsError` al arrancar. Corregido en `.env` y `.env.example`.

### 11.2 · Decisiones de diseño

- **ChromaDB sin contenedor propio:** el backend usa `chromadb.PersistentClient` (disco local), no HTTP client. Un volumen nombrado es suficiente — añadir un contenedor `chromadb/chroma` sería overhead innecesario.
- **Migraciones en startup:** `alembic upgrade head` en el `command` de compose garantiza que el schema esté actualizado cada vez que el backend arranca contra Postgres. Idempotente por diseño de Alembic.
- **Secretos vía `env_file`:** `backend/.env` provee `GEMINI_API_KEY`, `JWT_SECRET`, `AES_KEY`. Los overrides de `environment` solo afectan las variables que cambian entre SQLite-local y Postgres-compose.

### 11.3 · Cómo usarlo

```bash
# Levantar stack completo (primera vez o después de cambios en código)
docker compose up --build

# Levantar sin rebuild
docker compose up

# Detener preservando datos
docker compose down

# Detener y borrar volúmenes (reset completo)
docker compose down -v
```
